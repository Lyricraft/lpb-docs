# 开始

要使用 LPB 制作整合包，我们可以通过以下步骤进行。点击相应链接跳转到对应章节，了解这个步骤的详细信息。

在所有流程介绍完毕后，我们会给出一个 [基本构建示例](a-simple-example)，争取用最简单的语言来说明如何使用 LPB 来构建一个简单的整合包。如果你觉得下面的文档晦涩难懂，不妨先看看这一部分，它更易理解也更具趣味。将这个示例与一下正文结合来理解，或许能让你更快地上手。

## 安装

前往 [安装](setup.md) 章节，了解安装 LPB 不同方式，以及基本的配置和使用方法。

## 编辑配置文件

LPB 使用 YAML 格式的配置文件来定义整合包内容，因此，编辑配置文件是制作整合包至关重要的第一步。在此阶段，你会把你整合包的名字、支持的 Minecraft 版本和加载器，以及你想在其中包含哪些模组告诉 LPB，便于 LPB 之后帮你构建整合包。

要学会编辑配置文件，我们首先需要对 YAML 语法有一定的了解。YAML 官方文档（[YAML 1.2](https://yaml.org/spec/1.2/)）有详细的说明。当然，这里也推荐 [菜鸟教程](https://www.runoob.com/w3cnote/yaml-intro.html)，它是用中文写的，内容相对简单，也易于理解。我们对 YAML 的了解无须特别深入，知道对象、数组、键值对、数据类型的概念和写法就够了。

如果你不想要专门学习相关语法，本文档中会给足够的例子，我们也可以直接参照这些例子来配置自己的整合包。

现在，我们可以前往 [整合包配置总览](pack-configs/overview.md) 章节，了解配置文件的结构和内容，以及如何编辑它来定义你的整合包。

## 执行构建脚本

在配置好整合包后，我们还需要稍微编辑一下构建器的配置，然后运行构建脚本。这时 LPB 便会真正发挥作用，阅读我们的配置，搜集资源信息，生成数据，然后构建并打包，得到我们想要的整合包。

要了解如何运行构建脚本，我们可以前往 [构建脚本](build-a-pack/build-scripts.md) 章节，了解构建脚本的配置和使用方法，以及如何与脚本交互，和解决常见问题。
